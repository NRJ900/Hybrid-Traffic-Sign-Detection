# Utility Scripts Overview (`Codes/utils/`)

This directory contains the foundational, low-level helper scripts required to run, train, and mathematically evaluate the YOLOv5 architecture.

### `general.py`
The core mathematical and system utility module.
*   **Bounding Box Math**: Contains functions like `xyxy2xywh` and `xywh2xyxy` to rapidly convert bounding box coordinate systems.
*   **Coordinate Scaling**: The `scale_coords` function is heavily used by the Hybrid App to scale AI tensor coordinates back up to your original 1080p WebRTC/Video frames.
*   **Non-Maximum Suppression**: Houses the crucial `non_max_suppression` algorithm which deletes overlapping/redundant boxes generated by YOLO.
*   **System/File Ops**: Tools for logging, reproducibility (fixing seeds), and parsing `.yaml` hyperparameter configurations.

### `datasets.py`
The data-loading backbone for inference and model training.
*   **`LoadImages` & `LoadStreams`**: Classes designed to ingest local photos, raw `.mp4` video files, or active RTSP/WebRTC streams frame-by-frame. 
*   **`letterbox`**: Automatically pads and resizes incoming video arrays into square chunks (e.g., 640x640) that are uniformly digestible by neural networks, maintaining the original aspect ratio to prevent warping.
*   **Data Augmentation**: Functions for mosaic generation, HSV color shifting, perspective warping, and random rotations used exclusively during the `train.py` loop to artificially expand dataset variance.

### `torch_utils.py`
Hardware and PyTorch architecture optimization tools.
*   **`select_device`**: Automatically detects if an NVIDIA GPU (CUDA) is available on the system. If so, it funnels all floating-point math through the GPU; otherwise, it silently defaults back to CPU execution to prevent crashes.
*   **Pruning & Benchmarking**: Low-level model profiling metrics, allowing the developer to see exactly how many parameters and parameters gradients the active Neural Network holds.
*   **Model EMA**: Implements Exponential Moving Average over the model weights for more stable training convergence.

### `plots.py`
Visualization and graphing scripts.
*   **`plot_one_box`**: Contains the OpenCV `cv2.rectangle` bindings that physically draw the colored lines and text labels onto the ultimate output image array.
*   **Training Metrics**: Generates the graphs (loss curves, F1 scores, mAP@0.5 charts) that are spat out into the `runs/` folder at the conclusion of a training loop to visualize accuracy over time.

### `metrics.py` & `loss.py`
The mathematical graders for the AI.
*   **`metrics.py`**: Calculates the Intersection over Union (IoU), Precision, Recall, and Mean Average Precision.
*   **`loss.py`**: The backpropagation grader. It calculates the BCE (Binary Cross Entropy) and GIoU loss algorithms so the neural weights know exactly how "wrong" they were during a training pass.

### `autoanchor.py`
* Calculates the optimal "starter" bounding boxes using K-Means clustering across a custom dataset to help YOLO lock onto signs faster.

### `activations.py`
* Contains optimized Swish, Mish, and Hardswish activation functions for the deep neural layers.
### `google_utils.py`
* Cloud interactions for downloading pre-trained `.pt` files automatically from Ultralytics GCP buckets if they are missing locally.
